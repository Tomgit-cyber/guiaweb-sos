🎯 Objetivo

Este assistente serve como guia completo para desenvolvimento, manutenção e evolução do aplicativo SOS Unimed Vidas, incorporando conhecimento de múltiplas fontes e fornecendo orientação passo a passo.

O guia foi projetado para facilitar a transição para a equipe de TI da Unimed Cachoeiro do Itapemirim, garantindo continuidade do projeto após entrega do MVP.

🧰 Ferramentas Utilizadas

GitHub + Vercel ▼

Configuração Essencial

Repositório: github.com/unimedci/sos-vidas

Deploy Automático: Ativado via webhooks do Vercel

Passo a Passo para Correções

  1. Clonar repositório: git clone https://github.com/unimedci/sos-vidas.git
  2. Criar branch: git checkout -b fix/nome-da-correcao
  3. Fazer alterações necessárias
  4. Commit: git commit -m "fix: descrição breve"
  5. Push: git push origin fix/nome-da-correcao
  6. Abrir Pull Request no GitHub

🔄 Assistente de Transição

🔑 Fluxo de Entrada para Novos Devs

Primeira Semana na Equipe ▼
  1. Solicitar acessos ao gestor de TI
  2. Configurar ambiente local (seção 2 deste guia)
  3. Revisar documentação técnica (/docs)
  4. Fazer deploy de teste em staging
  5. Participar de reunião de onboarding

🛠️ Manutenção Rotineira

Atualização Mensal ▼
  1. Atualizar dependências: npm update
  2. Executar testes: npm run test
  3. Verificar logs de erro no Firebase
  4. Revisar métricas de performance no Vercel
  5. Atualizar documentação se necessário

🚨 Resolução de Problemas Comuns

Deploy Falhando ▼
  1. Verificar logs no Vercel
  2. Testar localmente: npm run dev
  3. Reverter para commit estável se necessário
  4. Corrigir erros em branch separada
  5. Testar novamente antes de merge
