🎯 Bem-vindo ao Guia de Desenvolvimento

Este guia foi criado para te orientar em cada etapa do desenvolvimento do aplicativo SOS Unimed Vidas, desde a configuração inicial até a entrega final do MVP.

Como usar este guia:

  1. Use o menu à esquerda para navegar pelas fases do desenvolvimento
  2. Clique em cada item para ver instruções detalhadas
  3. Siga os passos na ordem recomendada para melhor resultado
  4. Marque as tarefas como concluídas conforme você avança

O objetivo é entregar um MVP completo e funcional, com todas as etapas documentadas para facilitar a continuidade do projeto pela equipe de TI da Unimed.

Começar pelo Primeiro Passo

⚙️ Configuração no ChromeOS/Linux

Siga este guia para configurar seu ambiente de desenvolvimento Flutter no ChromeOS com suporte a Linux (Crostini).

📋 Pré-requisitos

  • Chromebook com suporte a Linux (Crostini)
  • Pelo menos 8GB de armazenamento livre
  • Conexão estável com a internet

🔧 Passo 1: Configurar o Ambiente Linux

Ativar Linux (Crostini)

  1. Acesse Configurações > Linux (Desenvolvedor)
  2. Clique em Ativar
  3. Siga as instruções na tela (pode demorar alguns minutos)

Atualizar pacotes

Abra o Terminal Linux e execute:

sudo apt-get update && sudo apt-get upgrade -y

📥 Passo 2: Instalar o Flutter

Baixar o Flutter SDK

cd ~
wget https://storage.googleapis.com/flutter_infra_release/releases/stable/linux/flutter_linux_3.22.0-stable.tar.xz

Extrair o arquivo

tar xf flutter_linux_3.22.0-stable.tar.xz

Adicionar ao PATH

  1. Edite o arquivo .bashrc:
    2. nano ~/.bashrc
  2. Adicione no final:
    3. export PATH="$PATH:$HOME/flutter/bin"
  3. Salve (Ctrl+O, Enter) e saia (Ctrl+X)
  4. Carregue as alterações:
    5. source ~/.bashrc

🔍 Passo 3: Verificar Dependências

Executar flutter doctor

flutter doctor

Você verá mensagens sobre dependências faltando - vamos resolver isso agora.

🤖 Passo 4: Configurar Android

Instalar Java JDK

sudo apt-get install openjdk-11-jdk -y

Baixar Android Studio

  1. Acesse developer.android.com/studio
  2. Baixe a versão para Linux
  3. No terminal, navegue até a pasta de downloads e extraia:
    4. cd ~/Downloads
    unzip android-studio-*-linux.tar.gz
    mv android-studio ~

Iniciar Android Studio

cd ~/android-studio/bin
./studio.sh

Siga o assistente de instalação e marque:

  • Android SDK
  • Android SDK Platform
  • Android Virtual Device

Configurar variáveis de ambiente

  1. Edite novamente o .bashrc:
    2. nano ~/.bashrc
  2. Adicione:
    3. export ANDROID_HOME="$HOME/Android/Sdk"
    export PATH="$PATH:$ANDROID_HOME/tools"
    export PATH="$PATH:$ANDROID_HOME/platform-tools"
  3. Salve e carregue:
    4. source ~/.bashrc

✅ Passo 5: Finalizar Configuração

Aceitar licenças Android

flutter doctor --android-licenses

Pressione y para aceitar todas as licenças.

Verificar instalação

flutter doctor

Agora deverá mostrar todas as dependências como OK.

⚙️ Configuração no Windows

Siga este guia para configurar seu ambiente de desenvolvimento Flutter no Windows.

📋 Pré-requisitos

  • Windows 10 ou superior (64-bit)
  • Pelo menos 8GB de RAM
  • Pelo menos 4GB de espaço em disco livre
  • Conexão estável com a internet

📥 Passo 1: Instalar o Flutter SDK

Baixar o Flutter SDK

  1. Acesse docs.flutter.dev/get-started/install/windows
  2. Baixe o arquivo flutter_windows_x.x.x-stable.zip
  3. Extraia o arquivo em C:\src\flutter (crie a pasta se necessário)

Adicionar ao PATH

  1. Pressione Win + R, digite sysdm.cpl e pressione Enter
  2. Clique na aba Avançado
  3. Clique em Variáveis de Ambiente
  4. Na seção Variáveis do sistema, selecione Path e clique em Editar
  5. Clique em Novo e adicione: C:\src\flutter\bin
  6. Clique OK para fechar todas as janelas

🔍 Passo 2: Verificar Dependências

Executar flutter doctor

Abra o Prompt de Comando ou PowerShell e execute:

flutter doctor

Você verá mensagens sobre dependências faltando - vamos resolver isso agora.

🤖 Passo 3: Instalar Android Studio

Baixar Android Studio

  1. Acesse developer.android.com/studio
  2. Baixe a versão para Windows
  3. Execute o instalador e siga as instruções

Configurar Android Studio

  1. Abra o Android Studio
  2. No assistente de instalação, marque:
    • Android SDK
    • Android SDK Platform
    • Android Virtual Device
  3. Clique em Next e complete a instalação

📱 Passo 4: Configurar Emulador Android (Opcional)

Criar um dispositivo virtual

  1. No Android Studio, vá em Tools > Device Manager
  2. Clique em Create Device
  3. Selecione um dispositivo (ex: Pixel 4)
  4. Selecione uma imagem do sistema (API 30 ou superior recomendado)
  5. Clique em Finish

✅ Passo 5: Finalizar Configuração

Aceitar licenças Android

Execute no Prompt de Comando ou PowerShell:

flutter doctor --android-licenses

Pressione y para aceitar todas as licenças.

Verificar instalação

Execute novamente:

flutter doctor

Agora deverá mostrar todas as dependências como OK.

Marcar como Concluído

⚙️ Configuração no macOS

Siga este guia para configurar seu ambiente de desenvolvimento Flutter no macOS.

📋 Pré-requisitos

  • macOS (High Sierra ou superior)
  • Pelo menos 8GB de RAM
  • Pelo menos 4GB de espaço em disco livre
  • Conexão estável com a internet

📥 Passo 1: Instalar o Flutter SDK

Baixar o Flutter SDK

  1. Acesse docs.flutter.dev/get-started/install/macos
  2. Baixe o arquivo flutter_macos_arm64_x.x.x-stable.zip (para Apple Silicon) ou flutter_macos_x64_x.x.x-stable.zip (para Intel)
  3. Extraia o arquivo na sua pasta Applications ou em um diretório de sua escolha (ex: ~/development)

Adicionar ao PATH

  1. Determine qual shell você está usando: echo $SHELL
  2. Se for zsh (padrão no macOS Catalina e posteriores), edite ~/.zshrc. Se for bash, edite ~/.bash_profile.
  3. Adicione a seguinte linha (ajuste o caminho se necessário):
    4. export PATH="$PATH:`pwd`/flutter/bin"
  4. ou, se extraiu em ~/development:
    5. export PATH="$PATH:$HOME/development/flutter/bin"
  5. Salve o arquivo e execute source ~/.zshrc (ou source ~/.bash_profile) para carregar as alterações.

🔍 Passo 2: Verificar Dependências

Executar flutter doctor

Abra o Terminal e execute:

flutter doctor

Você verá mensagens sobre dependências faltando - vamos resolver isso agora.

🍎 Passo 3: Configurar Xcode (para iOS)

Instalar Xcode

  1. Abra a App Store
  2. Procure por "Xcode" e instale
  3. Após a instalação, abra o Xcode e aceite os termos
  4. Execute no Terminal:
    5. sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer
    sudo xcodebuild -runFirstLaunch

🤖 Passo 4: Instalar Android Studio (para Android)

Baixar Android Studio

  1. Acesse developer.android.com/studio
  2. Baixe a versão para macOS
  3. Abra o arquivo .dmg e arraste o Android Studio para a pasta Applications
  4. Abra o Android Studio e siga o assistente de instalação

📱 Passo 5: Configurar Emulador Android (Opcional)

Criar um dispositivo virtual

  1. No Android Studio, vá em Tools > Device Manager
  2. Clique em Create Device
  3. Selecione um dispositivo (ex: Pixel 4)
  4. Selecione uma imagem do sistema (API 30 ou superior recomendado)
  5. Clique em Finish

✅ Passo 6: Finalizar Configuração

Aceitar licenças Android

Execute no Terminal:

flutter doctor --android-licenses

Pressione y para aceitar todas as licenças.

Verificar instalação

Execute novamente:

flutter doctor

Agora deverá mostrar todas as dependências como OK.

📞 Passo 1: Funcionalidade de Emergência

Ação do botão de emergência

O botão deve:

  1. Mostrar confirmação antes de chamar

// Exemplo de chamada telefônica
_makeEmergencyCall() async {
  const phoneNumber = 'tel:123456789';
  if (await launch(phoneNumber)) {
    // Registrar chamada
  } else {
    throw 'Could not launch $phoneNumber';
  }
}

Próximo: Integração Firebase

🔥 Integração com Firebase

Configurar Firebase para autenticação, banco de dados e analytics.

⚙️ Passo 1: Configurar Projeto Firebase

Criar projeto no Firebase Console

  1. Acesse Firebase Console
  2. Clique em "Criar projeto"
  3. Nomeie como "SOS Unimed Vidas"
  4. Ative Google Analytics (recomendado)

📱 Passo 2: Adicionar App Flutter

Configurar para Android

  1. No Firebase Console, adicione um app Android
  2. Use o package name do seu app Flutter
  3. Baixe o arquivo google-services.json
  4. Coloque na pasta android/app/

📦 Passo 3: Adicionar Dependências

Atualizar pubspec.yaml

dependencies:
  firebase_core: ^2.0.0
  firebase_auth: ^4.0.0
  cloud_firestore: ^4.0.0
  firebase_analytics: ^10.0.0

🔌 Passo 4: Inicializar Firebase

Configurar inicialização

No main.dart:

import 'package:firebase_core/firebase_core.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await Firebase.initializeApp();
  runApp(MyApp());
}

🔐 Passo 5: Implementar Autenticação

Fluxo de login básico

Implementar tela de login com CPF e senha:

// Exemplo de autenticação
final FirebaseAuth _auth = FirebaseAuth.instance;

Future signIn(String cpf, String password) async {
  try {
    UserCredential result = await _auth.signInWithEmailAndPassword(
      email: '$cpf@unimed.com',
      password: password,
    );
    return result.user;
  } catch (e) {
    print(e.toString());
    return null;
  }
}

Próximo: Testes Unitários

🌐 Integração com API Externa

Configurar integração com APIs externas para funcionalidades adicionais.

🔌 Passo 1: Configurar Cliente HTTP

Adicionar dependência

Adicione ao pubspec.yaml:

dependencies:
  http: ^0.13.5

📡 Passo 2: Criar Serviço de API

Exemplo de serviço

Crie lib/services/api_service.dart:

import 'dart:convert';
import 'package:http/http.dart' as http;

class ApiService {
  static const String baseUrl = 'https://api.example.com';

  Future> fetchData(String endpoint) async {
    final response = await http.get(Uri.parse('$baseUrl/$endpoint'));

    if (response.statusCode == 200) {
      return json.decode(response.body);
    } else {
      throw Exception('Falha ao carregar dados');
    }
  }
}

🔒 Passo 3: Gerenciar Autenticação

Headers de autenticação

Adicione headers para APIs que requerem autenticação:

Future getDataWithAuth(String token) async {
  return await http.get(
    Uri.parse('https://api.example.com/data'),
    headers: {
      'Authorization': 'Bearer $token',
      'Content-Type': 'application/json',
    },
  );
}

🔄 Passo 4: Tratar Erros

Implementar tratamento de erros

Adicione tratamento para diferentes códigos de status:

try {
  final response = await http.get(Uri.parse(url));
  if (response.statusCode == 200) {
    // Sucesso
  } else if (response.statusCode == 401) {
    // Não autorizado
  } else if (response.statusCode == 404) {
    // Não encontrado
  } else {
    // Outro erro
  }
} catch (e) {
  // Erro de conexão
}

🔐 Passo 3: Configurar Signing (Android)

Criar keystore para assinatura

No terminal:

keytool -genkey -v -keystore ~/upload-keystore.jks -keyalg RSA -keysize 2048 -validity 10000 -alias upload

Referencie o keystore no arquivo android/key.properties:

storePassword=
keyPassword=
keyAlias=upload
storeFile=/Users//upload-keystore.jks
