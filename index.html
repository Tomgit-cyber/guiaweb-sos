📘 Visão Geral do Projeto

Este documento apresenta a visão macro do projeto, detalhando seus objetivos, escopo e principais componentes.

🎯 Objetivo Principal

Desenvolver um aplicativo móvel de código aberto e multiplataforma que permita aos associados da Unimed acionar rapidamente o serviço de emergência.

📱 Escopo do MVP

  • Funcionalidade Chave: Botão de emergência para ligar para a Unimed.
  • Autenticação: Login seguro para associados.
  • Perfil do Usuário: Visualização de dados básicos.
  • Histórico de Chamadas: Registro das chamadas realizadas.

⚙️ Configuração no ChromeOS/Linux

Siga este guia para configurar seu ambiente de desenvolvimento Flutter no ChromeOS com suporte a Linux (Crostini).

Passo 1: Ativar e Atualizar Linux

Acesse Configurações > Desenvolvedor > Ambiente de desenvolvimento Linux e ative. No terminal, execute:

sudo apt-get update && sudo apt-get upgrade -y

Passo 2: Instalar o Flutter SDK

Baixe o SDK, extraia e adicione ao seu PATH em ~/.bashrc ou ~/.zshrc.

export PATH="$PATH:[CAMINHO_PARA_FLUTTER]/flutter/bin"

Passo 3: Flutter Doctor

Execute flutter doctor e siga as instruções para instalar dependências como o Android SDK e Command-line Tools.

⚙️ Configuração no Windows

Passo 1: Instalar Flutter SDK

Baixe o zip do SDK do Flutter. Extraia em C:\src\flutter.

Passo 2: Atualizar o PATH

Adicione C:\src\flutter\bin às "Variáveis de Ambiente" do sistema.

Passo 3: Instalar Android Studio

Instale o Android Studio. Ele instalará o Android SDK. Depois, rode flutter doctor e aceite as licenças.

Marcar como Concluído

⚙️ Configuração no macOS

Passo 1: Instalar Flutter SDK

Baixe o SDK, extraia (ex: `~/dev/flutter`) e adicione ao seu PATH em `~/.zshrc`.

Passo 2: Instalar Xcode e Android Studio

Instale o Xcode pela App Store. Instale o Android Studio para as ferramentas Android.

Passo 3: Flutter Doctor e Cocoapods

Rode flutter doctor. Se necessário, instale o Cocoapods com sudo gem install cocoapods.

🤖 Instalação do Android Studio

Download e Instalação

Visite o site oficial do Android, baixe o instalador e siga o assistente. Certifique-se de que "Android SDK", "Android SDK Platform" e "Android Virtual Device (AVD)" estejam selecionados.

Marcar como Concluído

📱 Gerenciar SDKs do Android

Acessar o SDK Manager

No Android Studio, vá em `File > Settings > Appearance & Behavior > System Settings > Android SDK`.

Instalar Plataformas e Ferramentas

Na aba "SDK Platforms", instale pelo menos uma versão recente (ex: API 33). Na aba "SDK Tools", garanta que "Android SDK Build-Tools", "Android Emulator" e "Android SDK Platform-Tools" estejam instalados.

Marcar como Concluído

📱 Criar Emulador (AVD)

Acessar o Device Manager

Em `Tools > Device Manager`, clique em "Create Device".

Configurar o Emulador

Escolha um hardware (ex: Pixel 6), uma imagem de sistema (ex: API 33) e finalize a criação. Inicie o emulador clicando no ícone de play.

🔥 Integração com Firebase

Configurar Projeto e App

Crie o projeto no console do Firebase e adicione um app Android/iOS. Baixe e posicione o arquivo `google-services.json`.

Adicionar Dependências

Adicione `firebase_core`, `firebase_auth` e `cloud_firestore` ao `pubspec.yaml`.

Implementar Autenticação

Crie o fluxo de login usando o `FirebaseAuth.instance.signInWithEmailAndPassword()`.

🔮 Automações Futuras (n8n)

O que é n8n?

É uma ferramenta de automação de código aberto para conectar diferentes APIs e serviços.

Casos de Uso

Poderia ser usado para, ao acionar a emergência, automaticamente notificar equipes por Slack, registrar em uma planilha Google e enviar um SMS para um administrador.
