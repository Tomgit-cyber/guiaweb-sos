🎯 Bem-vindo ao Guia de Desenvolvimento

ℹ️ Sobre este Guia

Este guia é específico para o desenvolvimento do aplicativo SOS Unimed Vidas utilizando o framework Flutter. Ele fornece instruções passo a passo para configurar o ambiente, criar o projeto, implementar funcionalidades e preparar o MVP para entrega.

Escopo: O foco está em criar um aplicativo multiplataforma (Android, iOS, Web) com Flutter. Este NÃO é um guia geral de desenvolvimento web ou mobile. As instruções são práticas e diretas ao objetivo do projeto.

Este guia orienta você em cada etapa do desenvolvimento do aplicativo SOS Unimed Vidas, desde a configuração inicial até a entrega do MVP funcional.

Como usar este guia:

  1. Use o menu à esquerda para navegar pelas fases do desenvolvimento.
  2. Clique em cada item para ver instruções detalhadas.
  3. Consulte a seção Solução de Problemas caso encontre dificuldades.
  4. Siga os passos na ordem para melhores resultados.
  5. Marque as tarefas como concluídas para acompanhar seu progresso.

O objetivo é criar um aplicativo funcional com documentação clara para a equipe de TI da Unimed continuar o projeto.

🖥️ Configuração Inicial: Windows

Vamos configurar seu ambiente de desenvolvimento Flutter no Windows.

📥 Passo 1: Baixar e Extrair Flutter SDK

Baixar Flutter SDK

Baixar Flutter SDK para Windows

Importante: Baixe a versão estável mais recente.

Extrair SDK

  1. Extraia o arquivo ZIP baixado para um local conhecido, como D:\flutter ou C:\src\flutter.
  2. Evite extrair para pastas com espaços ou caracteres especiais no nome (ex: "Arquivos de Programas").
  3. Após a extração, verifique se o arquivo flutter\bin\flutter.bat existe no local escolhido.

⚙️ Passo 2: Configurar PATH do Sistema

Adicionar Flutter ao PATH

  1. Pressione Win + R, digite sysdm.cpl e pressione Enter.
  2. Na aba Avançado, clique em Variáveis de Ambiente.
  3. Em Variáveis do sistema, selecione Path e clique em Editar.
  4. Clique em Novo e adicione o caminho completo para a pasta bin do Flutter (ex: D:\flutter\bin).
  5. Clique em OK para salvar todas as alterações.

Nota: Feche e reabra o terminal (Prompt de Comando ou PowerShell) para aplicar as mudanças.

🔍 Passo 3: Verificar Dependências

Executar flutter doctor

Abra um novo terminal e execute:

flutter doctor

Se aparecer um erro como "'flutter' não é reconhecido", vá para Solução de Problemas.

🤖 Passo 4: Instalar Android Studio

Baixar Android Studio

Baixar Android Studio

Importante: O Android Studio é necessário para desenvolver apps Android.

Instalar Android Studio

  1. Execute o instalador baixado.
  2. Siga as instruções na tela.
  3. Durante a instalação, certifique-se de marcar as opções para instalar o Android SDK, Android SDK Platform-Tools e Android SDK Build-Tools.
  4. Após a instalação, inicie o Android Studio.
  5. Siga o assistente de configuração para instalar as plataformas Android necessárias (SDK).

📜 Passo 5: Aceitar Licenças do Android

Aceitar licenças

Execute no terminal:

flutter doctor --android-licenses

Pressione y para aceitar cada licença solicitada.

✅ Passo 6: Verificar Instalação

Confirmar configuração

Execute novamente:

flutter doctor

Confirme que todas as dependências estão marcadas como OK.

Próximo: Lógica de Negócio

🧠 Lógica de Negócio

Implemente a lógica para enviar chamadas de emergência.

🔌 Passo 1: Criar Serviço de Emergência

Implementar EmergencyService

Crie lib/services/emergency_service.dart:

import 'package:http/http.dart' as http;
import 'dart:convert';

class EmergencyService {
  final String baseUrl = 'https://api.exemplo.com'; // Substitua pela URL real

  Future<void> sendEmergencyRequest() async {
    final url = Uri.parse('$baseUrl/emergency');
    final response = await http.post(
      url,
      headers: {'Content-Type': 'application/json'},
      body: jsonEncode({
        'timestamp': DateTime.now().toIso8601String(),
        'location': 'Localização do usuário', // Implementar geolocalização
      }),
    );

    if (response.statusCode != 200) {
      throw Exception('Falha ao enviar emergência: ${response.statusCode}');
    }
  }
}

🌐 Integração com API Externa

Integre uma API externa para funcionalidades adicionais.

📋 Passo 1: Configurar HTTP

Usar Pacote HTTP

Certifique-se de que http está no pubspec.yaml. Crie lib/services/api_service.dart:

import 'package:http/http.dart' as http;
import 'dart:convert';

class ApiService {
  Future<Map<String, dynamic>> fetchData() async {
    final url = Uri.parse('https://api.exemplo.com/data');
    final response = await http.get(url);

    if (response.statusCode == 200) {
      return jsonDecode(response.body);
    } else {
      throw Exception('Falha ao carregar dados');
    }
  }
}

📱 Passo 2: Usar na Tela

Consumir dados na interface

Exemplo em uma tela:

import 'package:flutter/material.dart';
import '../services/api_service.dart';

class DataScreen extends StatelessWidget {
  final ApiService apiService = ApiService();

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('SOS Unimed Vidas'),
        backgroundColor: Theme.of(context).primaryColor,
      ),
      body: FutureBuilder<Map<String, dynamic>>(
        future: apiService.fetchData(),
        builder: (context, snapshot) {
          if (snapshot.connectionState == ConnectionState.waiting) {
            return Center(child: CircularProgressIndicator());
          } else if (snapshot.hasError) {
            return Center(child: Text('Erro ao carregar dados'));
          } else {
            return Center(
              child: Text(
                'Dados: ${snapshot.data.toString()}',
                style: TextStyle(color: Colors.white),
              ),
            );
          }
        },
      ),
    );
  }
}

'flutter' não é reconhecido como um comando interno ou externo, um programa operável ou um arquivo em lotes.

Como Resolver

  1. Confirme que o Flutter SDK está no local correto (ex: D:\flutter).
  2. Verifique se o arquivo flutter.bat existe em D:\flutter\bin\.
  3. Verifique o PATH:
    • Pressione Win + R, digite sysdm.cpl, Enter.
    • Aba Avançado > Variáveis de Ambiente.
    • Em Variáveis do sistema, edite Path.
    • Adicione D:\flutter\bin se não estiver presente.
    • Salve com OK.
  4. Feche e reabra o terminal.
  5. Execute:
    flutter doctor

Nota: Se persistir, verifique permissões na pasta D:\flutter ou execute o terminal como administrador.

macOS

Como Resolver

  1. Confirme que o Flutter SDK está em ~/development/flutter.
  2. Edite ~/.zshrc:
    nano ~/.zshrc
  3. Adicione:
    export PATH="$PATH:~/development/flutter/bin"
  4. Salve e execute:
    source ~/.zshrc
  5. Teste:
    flutter doctor

Linux/ChromeOS

Como Resolver

  1. Confirme que o Flutter SDK está em ~/development/flutter.
  2. Edite ~/.bashrc:
    nano ~/.bashrc
  3. Adicione:
    export PATH="$PATH:~/development/flutter/bin"
  4. Salve e execute:
    source ~/.bashrc
  5. Teste:
    flutter doctor

❓ Solução de Problemas: Erro de Licenças Android

Se o flutter doctor indicar que as licenças Android não estão aceitas:

Como Resolver

  1. Execute:
    flutter doctor --android-licenses
  2. Pressione y para aceitar todas as licenças.
  3. Verifique:
    flutter doctor

❓ Solução de Problemas: Problemas com Emulador

Se o emulador Android não iniciar ou não for detectado:

Como Resolver

  1. Abra o Android Studio.
  2. Vá para Configure > AVD Manager.
  3. Crie ou inicie um emulador Android.
  4. Verifique se o emulador aparece em:
    flutter devices
  5. Se não aparecer, reinicie o ADB:
    adb kill-server && adb start-server
