📘 Visão Geral do Projeto

Este documento apresenta a visão macro do projeto, detalhando seus objetivos, escopo e principais componentes.

🎯 Objetivo Principal

Desenvolver um aplicativo móvel de código aberto e multiplataforma que permita aos associados da Unimed acionar rapidamente o serviço de emergência, facilitando o acesso a cuidados médicos urgentes.

📱 Escopo do MVP

  • Funcionalidade Chave: Botão de emergência para ligar diretamente para o serviço da Unimed.
  • Autenticação: Login seguro para associados (CPF ou código de beneficiário).
  • Perfil do Usuário: Visualização de dados básicos e informações médicas essenciais.
  • Histórico de Chamadas: Registro das chamadas de emergência realizadas pelo app.

🛠️ Stack de Tecnologia

  • Frontend (Aplicativo): Flutter (Framework Dart multiplataforma).
  • Backend & Infraestrutura: Firebase (Authentication, Firestore, Analytics).
  • IDE Recomendada: Visual Studio Code ou Android Studio.

⚙️ Configuração no ChromeOS/Linux

Siga este guia para configurar seu ambiente de desenvolvimento Flutter no ChromeOS com suporte a Linux (Crostini) ou em uma distribuição Linux padrão.

Passo 1: Ativar Linux e Atualizar

No ChromeOS, acesse Configurações > Desenvolvedor > Ambiente de desenvolvimento Linux e ative. Depois, no terminal (ou em qualquer distro Linux), execute:

sudo apt-get update && sudo apt-get upgrade -y

Passo 2: Instalar Flutter SDK

Baixe o SDK do site oficial do Flutter, extraia-o para uma pasta (ex: `~/development/flutter`) e adicione ao seu PATH editando o arquivo ~/.bashrc ou ~/.zshrc.

export PATH="$PATH:$HOME/development/flutter/bin"

Lembre-se de recarregar o terminal ou rodar source ~/.bashrc.

Passo 3: Executar Flutter Doctor

Este comando verifica se há dependências faltando e te guia na instalação delas (como o Android SDK e Android Studio).

flutter doctor

Siga as instruções para resolver os problemas apontados, especialmente a aceitação das licenças do Android com flutter doctor --android-licenses.

⚙️ Configuração no Windows

Configurando o ambiente de desenvolvimento Flutter no Windows.

Passo 1: Instalar Flutter SDK

Baixe o zip do SDK do site do Flutter. Extraia em um local sem espaços ou privilégios especiais (ex: C:\src\flutter). NÃO use C:\Program Files\.

Passo 2: Atualizar o PATH

Adicione o caminho completo para C:\src\flutter\bin às suas "Variáveis de Ambiente" do sistema.

Passo 3: Instalar Android Studio e Executar Doctor

Instale o Android Studio. Ele instalará o Android SDK necessário. Depois, abra um novo terminal e rode flutter doctor, seguindo as instruções para resolver qualquer pendência.

Marcar como Concluído
Próximo: macOS

⚙️ Configuração no macOS

Configurando o ambiente de desenvolvimento Flutter no macOS.

Passo 1: Instalar Flutter SDK

Baixe o zip do SDK. Extraia para uma pasta (ex: `~/development/flutter`) e adicione ao seu PATH editando `~/.zshrc` (padrão) ou `~/.bash_profile`.

export PATH="$PATH:$HOME/development/flutter/bin"

Passo 2: Instalar Xcode e Android Studio

Instale o Xcode pela App Store. Execute-o uma vez para aceitar os termos. Instale o Android Studio para ter as ferramentas de desenvolvimento Android.

Passo 3: Executar Doctor

Abra um novo terminal e rode flutter doctor. Siga as instruções para configurar o Cocoapods (se necessário) e aceitar as licenças do Android.

Próximo: Criar Projeto

💻 Criar o Projeto Flutter

Com o ambiente configurado, vamos criar a base do nosso aplicativo.

Executar o comando de criação

No terminal, navegue até sua pasta de projetos e execute o comando abaixo, seguindo as convenções de nomenclatura (letras minúsculas e underscores).

flutter create sos_unimed_vidas

Abrir no Editor e Executar

Navegue para a nova pasta com cd sos_unimed_vidas. Abra o projeto no seu editor (VS Code ou Android Studio). Inicie um emulador ou conecte um dispositivo e execute o app pela primeira vez (com F5 no VS Code ou clicando no botão "Run").

Você deverá ver o aplicativo de exemplo do contador.

Próximo: Estrutura Básica

🏗️ Estrutura Básica do Aplicativo

Vamos organizar nosso código para que ele seja escalável e fácil de manter.

Criar Estrutura de Pastas

Dentro da pasta lib/, crie uma estrutura de pastas para organizar os arquivos. Uma boa prática comum é:

  • lib/src/features - Para cada funcionalidade principal (ex: auth, home, profile).
  • lib/src/shared - Para componentes e serviços compartilhados (ex: widgets, models, services).
  • lib/main.dart - Ponto de entrada da aplicação.

Configurar Rotas

Defina a navegação principal do seu aplicativo. Use um gerenciador de rotas como o GoRouter para uma navegação mais robusta e baseada em URLs.

🎨 Interface do Usuário (UI)

Construindo a aparência do aplicativo, seguindo a identidade visual da Unimed.

Definir o Tema Global

No `MaterialApp` dentro de `main.dart`, defina um `ThemeData` global com as cores primárias, fontes e estilos de botões da Unimed.

Criar Widgets Reutilizáveis

Crie widgets customizados para elementos que se repetem, como botões principais, cards de informação e app bars personalizadas. Coloque-os na pasta de widgets compartilhados.

Próximo: Firebase

🔥 Integração com Firebase

Conectando nosso aplicativo aos serviços de backend do Firebase.

Configurar o Projeto no Firebase

Crie um projeto no Console do Firebase. Ative os serviços de Authentication, Firestore e Analytics. Siga o assistente para adicionar um aplicativo Android e/ou iOS ao projeto.

Adicionar FlutterFire

Use a CLI do FlutterFire (`flutterfire configure`) para conectar seu app ao Firebase automaticamente. Isso irá gerar o arquivo de inicialização necessário.

Implementar Autenticação

Crie o fluxo de login usando o Firebase Authentication. Permita o login com e-mail/senha ou telefone.
