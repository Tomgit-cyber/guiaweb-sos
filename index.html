🎯 Bem-vindo ao Guia de Desenvolvimento

Este guia foi criado para te orientar em cada etapa do desenvolvimento do aplicativo SOS Unimed Vidas, desde a configuração inicial até a entrega final do MVP.

Como usar este guia:

  1. Use o menu à esquerda para navegar pelas fases do desenvolvimento
  2. Clique em cada item para ver instruções detalhadas

⚙️ Configuração no Windows

Siga este guia para configurar seu ambiente de desenvolvimento Flutter no Windows.

📋 Pré-requisitos

  • Windows 10 ou superior (64-bit)
  • Pelo menos 8GB de RAM
  • Pelo menos 4GB de espaço em disco livre
  • Conexão estável com a internet

📥 Passo 1: Instalar o Flutter SDK

Baixar o Flutter SDK

Baixar Flutter SDK para Windows (v3.22.2)

O que é isso? Este é o kit de desenvolvimento principal do Flutter. Ele contém todas as ferramentas, bibliotecas e o compilador necessários para criar, testar e compilar o aplicativo. Para este guia, usaremos a versão estável mais recente para garantir acesso às últimas funcionalidades e correções de segurança.

Adicionar ao PATH

  1. Pressione Win + R, digite sysdm.cpl e pressione Enter
  2. Clique na aba Avançado
  3. Clique em Variáveis de Ambiente
  4. Na seção Variáveis do sistema, selecione Path e clique em Editar
  5. Clique em Novo e adicione: C:\src\flutter\bin
  6. Clique OK para fechar todas as janelas

Importante: Após realizar esta etapa, você precisa fechar e abrir novamente qualquer terminal (Prompt de Comando, PowerShell, etc.) para que a alteração tenha efeito.

🔍 Passo 2: Verificar Dependências

Executar flutter doctor

Abra um novo terminal e execute:

flutter doctor

Se você vir um erro de "comando não reconhecido", consulte a seção de Solução de Problemas.

🤖 Passo 3: Instalar Android Studio

Baixar Android Studio

Baixar Android Studio

Por que isso é necessário? O Android Studio é o ambiente de desenvolvimento oficial do Google para Android. Mesmo que você use outro editor de código (como o VS Code), sua instalação é **essencial**, pois ele fornece o Android SDK, as ferramentas de linha de comando e o gerenciador de emuladores que o Flutter utiliza para construir e testar o aplicativo para Android.

Configurar Android Studio

  1. Abra o Android Studio.
  2. No assistente de instalação, certifique-se de marcar:
    • Android SDK
    • Android SDK Platform
    • Android Virtual Device
  3. Clique em Next e complete a instalação.

✅ Passo 5: Finalizar Configuração

Aceitar licenças Android

Execute no Prompt de Comando ou PowerShell:

flutter doctor --android-licenses

Pressione y para aceitar todas as licenças.

Verificar instalação

Execute novamente:

flutter doctor

Agora deverá mostrar todas as dependências como OK.

❓ Solução de Problemas: Comando 'flutter' não reconhecido

Este é um dos erros mais comuns durante a configuração inicial e geralmente tem uma solução simples.

Cenário do Erro

Você abre o terminal (Prompt de Comando, PowerShell, etc.), digita flutter doctor e recebe a seguinte mensagem:

'flutter' não é reconhecido como um comando interno
ou externo, um programa operável ou um arquivo em lotes.

Causa Principal

O erro indica que o sistema operacional não sabe onde encontrar o executável do Flutter. Isso acontece porque o local de instalação do Flutter SDK não foi adicionado corretamente à variável de ambiente `Path` do seu sistema.

✅ Como Resolver

Siga estes passos para verificar e corrigir o problema:

  1. Verifique o Caminho do Flutter:
    Confirme em qual pasta você extraiu o Flutter SDK. No nosso guia, recomendamos C:\src\flutter. O caminho que precisa estar no Path é a subpasta bin dentro dela, ou seja: C:\src\flutter\bin.
  2. Corrija as Variáveis de Ambiente:
    • Pressione Win + R, digite sysdm.cpl e pressione Enter.
    • Vá para a aba Avançado e clique em Variáveis de Ambiente.
    • Na seção inferior ("Variáveis do sistema"), encontre a variável Path, selecione-a e clique em Editar.
    • Verifique se o caminho C:\src\flutter\bin está na lista. Se não estiver, clique em Novo e adicione-o.
    • Certifique-se de que não há erros de digitação.
    • Clique em OK em todas as janelas para salvar.
  3. Reinicie o Terminal (Passo Crucial!):
    As alterações nas variáveis de ambiente só são aplicadas a novas sessões do terminal. Feche todas as janelas do Prompt de Comando ou PowerShell e abra uma nova.
  4. Tente Novamente:
    No novo terminal, digite o comando novamente:
    flutter doctor
    O comando agora deve ser reconhecido e começar a executar a verificação do Flutter.
