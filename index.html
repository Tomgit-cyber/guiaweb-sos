🎯 Bem-vindo ao Guia de Desenvolvimento

Este guia orienta você em cada etapa do desenvolvimento do aplicativo SOS Unimed Vidas, desde a configuração inicial até a entrega do MVP funcional.

Como usar este guia:

  1. Use o menu à esquerda para navegar pelas fases do desenvolvimento.
  2. Clique em cada item para ver instruções detalhadas.
  3. Consulte a seção Solução de Problemas caso encontre dificuldades.
  4. Siga os passos na ordem para melhores resultados.
  5. Marque as tarefas como concluídas para acompanhar seu progresso.

O objetivo é criar um aplicativo funcional com documentação clara para a equipe de TI da Unimed continuar o projeto.

Começar pelo Primeiro Passo

⚙️ Configuração no Windows

Configure seu ambiente de desenvolvimento Flutter no Windows com este guia simples, ideal para iniciantes.

📋 Pré-requisitos

  • Windows 10 ou superior (64-bit)
  • Mínimo de 8GB de RAM
  • 4GB de espaço livre em disco
  • Conexão estável com a internet

📥 Passo 1: Instalar o Flutter SDK

Baixar o Flutter SDK

Baixar Flutter SDK para Windows

O que é isso? O Flutter SDK é a ferramenta principal para criar o aplicativo. Ele inclui tudo necessário para desenvolvimento, testes e build.

  1. Baixe o arquivo ZIP e extraia-o para C:\src\flutter.
  2. Evite pastas com permissões restritas, como C:\Program Files.

Adicionar ao PATH

  1. Pressione Win + R, digite sysdm.cpl e pressione Enter.
  2. Na aba Avançado, clique em Variáveis de Ambiente.
  3. Em Variáveis do sistema, selecione Path e clique em Editar.
  4. Clique em Novo e adicione: C:\src\flutter\bin.
  5. Clique em OK para salvar.

Nota: Feche e reabra o terminal (Prompt de Comando ou PowerShell) para aplicar as mudanças.

🔍 Passo 2: Verificar Dependências

Executar flutter doctor

Abra um novo terminal e execute:

flutter doctor

Se aparecer um erro como "'flutter' não é reconhecido", vá para Solução de Problemas.

🤖 Passo 3: Instalar Android Studio

Baixar Android Studio

Baixar Android Studio

Por que isso é necessário? O Android Studio fornece ferramentas para testar e construir o app no Android.

Configurar Android Studio

  1. Abra o Android Studio.
  2. No assistente de instalação, marque:
    • Android SDK
    • Android SDK Platform
    • Android Virtual Device
  3. Clique em Next e conclua a instalação.

✅ Passo 4: Finalizar Configuração

Aceitar licenças Android

Execute:

flutter doctor --android-licenses

Pressione y para aceitar todas as licenças.

Verificar instalação

Execute novamente:

flutter doctor

Confirme que todas as dependências estão marcadas como OK.

❓ Solução de Problemas: Comando 'flutter' não reconhecido

Este erro ocorre quando o sistema não encontra o Flutter. Vamos corrigir!

Windows

Cenário do Erro

Você digita flutter doctor e vê:

'flutter' não é reconhecido como um comando interno ou externo, um programa operável ou um arquivo em lotes.

Como Resolver

  1. Confirme que o Flutter SDK está em C:\src\flutter.
  2. Verifique o PATH:
    • Pressione Win + R, digite sysdm.cpl, Enter.
    • Aba Avançado > Variáveis de Ambiente.
    • Em Variáveis do sistema, edite Path.
    • Adicione C:\src\flutter\bin se não estiver presente.
    • Salve com OK.
  3. Feche e reabra o terminal.
  4. Execute:
    flutter doctor

Nota: Se persistir, verifique permissões na pasta C:\src\flutter ou execute o terminal como administrador.

macOS

Como Resolver

  1. Confirme que o Flutter SDK está em ~/development/flutter.
  2. Edite ~/.zshrc:
    nano ~/.zshrc
  3. Adicione:
    export PATH="$PATH:~/development/flutter/bin"
  4. Salve e execute:
    source ~/.zshrc
  5. Teste:
    flutter doctor

Linux/ChromeOS

Como Resolver

  1. Confirme que o Flutter SDK está em ~/flutter.
  2. Edite ~/.bashrc ou ~/.zshrc:
    nano ~/.bashrc
  3. Adicione:
    export PATH="$PATH:~/flutter/bin"
  4. Salve e execute:
    source ~/.bashrc
  5. Teste:
    flutter doctor

❓ Solução de Problemas: Problemas com Emulador

Se o emulador Android não iniciar ou não for detectado:

Como Resolver

  1. Abra o Android Studio > Configure > AVD Manager.
  2. Crie um dispositivo virtual (ex.: Pixel 4, API recente).
  3. Inicie o emulador e verifique:
    flutter devices
  4. Se não detectado, confirme o Android SDK no Android Studio.

Nota: Emuladores exigem bom desempenho. Considere um dispositivo físico se for lento.

Próximo: Estrutura Básica

📂 Estrutura Básica

Entenda a estrutura do projeto Flutter para organizar o código do SOS Unimed Vidas.

📋 Passo 1: Conhecer a Estrutura

Estrutura de Pastas

Após criar o projeto, a pasta sos_unimed_vidas contém:

  • lib/: Código-fonte principal (onde você vai trabalhar).
  • android/ e ios/: Configurações específicas para cada plataforma.
  • pubspec.yaml: Arquivo para gerenciar dependências.

📝 Passo 2: Configurar o pubspec.yaml

Adicionar Dependências

Edite pubspec.yaml para adicionar dependências básicas:

dependencies: flutter: sdk: flutter http: ^0.13.5 provider: ^6.0.5 firebase_core: ^2.4.0

Execute:

flutter pub get

Por que isso? http para chamadas de API, provider para gerenciar estado, e firebase_core para integração com Firebase.

🗂️ Passo 3: Organizar o Código

Criar Estrutura de Pastas

Crie na pasta lib/:

  • lib/screens/: Telas do aplicativo.
  • lib/services/: Lógica de negócio (ex.: chamadas de API).
  • lib/models/: Modelos de dados.
  • lib/widgets/: Componentes reutilizáveis.

⚙️ Lógica de Negócio

Adicione a lógica para a funcionalidade de emergência do SOS Unimed Vidas.

📋 Passo 1: Criar Serviço de Emergência

Implementar Serviço

Crie lib/services/emergency_service.dart:

import 'package:http/http.dart' as http; class EmergencyService { Future sendEmergencyRequest() async { try { final response = await http.post( Uri.parse('https://api.unimed.com/emergency'), // Substitua pela URL real body: {'user_id': '123', 'location': 'lat: -20.123, long: -40.456'}, ); if (response.statusCode == 200) { print('Emergência enviada com sucesso'); } else { throw Exception('Falha ao enviar emergência'); } } catch (e) { print('Erro: $e'); } } }

🌐 Integração com API Externa

Integre uma API externa para funcionalidades adicionais.

📋 Passo 1: Configurar HTTP

Usar Pacote HTTP

Certifique-se de que http está no pubspec.yaml. Crie lib/services/api_service.dart:

import 'package:http/http.dart' as http; import 'dart:convert'; class ApiService { Future> fetchData() async { final response = await http.get(Uri.parse('https://api.unimed.com/data')); // Substitua pela URL real if (response.statusCode == 200) { return jsonDecode(response.body); } else { throw Exception('Falha ao carregar dados'); } } }

🔗 Passo 2: Usar API na Tela

Atualizar HomeScreen

Edite lib/screens/home_screen.dart para exibir dados:

import 'package:flutter/material.dart'; import '../services/api_service.dart'; class HomeScreen extends StatelessWidget { final ApiService apiService = ApiService(); @override Widget build(BuildContext context) { return Scaffold( appBar: AppBar( title: Text('SOS Unimed Vidas'), backgroundColor: Theme.of(context).primaryColor, ), body: FutureBuilder>( future: apiService.fetchData(), builder: (context, snapshot) { if (snapshot.connectionState == ConnectionState.waiting) { return Center(child: CircularProgressIndicator()); } else if (snapshot.hasError) { return Center(child: Text('Erro ao carregar dados')); } else { return Center( child: Text( 'Dados: ${snapshot.data.toString()}', style: TextStyle(color: Colors.white), ), ); } }, ), ); } }
