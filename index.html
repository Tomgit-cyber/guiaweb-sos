🎯 Bem-vindo ao Guia de Desenvolvimento

Este guia foi criado para te orientar em cada etapa do desenvolvimento do aplicativo SOS Unimed Vidas, desde a configuração inicial até a entrega final do MVP.

Como usar este guia:

  1. Use o menu à esquerda para navegar pelas fases do desenvolvimento
  2. Clique em cada item para ver instruções detalhadas
  3. Se encontrar um problema, verifique a seção Solução de Problemas.
  4. Siga os passos na ordem recomendada para melhor resultado
  5. Marque as tarefas como concluídas conforme você avança

O objetivo é entregar um MVP completo e funcional, com todas as etapas documentadas para facilitar a continuidade do projeto pela equipe de TI da Unimed.

Começar pelo Primeiro Passo

⚙️ Configuração no Windows

Siga este guia para configurar seu ambiente de desenvolvimento Flutter no Windows. Ele é feito para ser simples, mesmo para iniciantes!

📋 Pré-requisitos

  • Windows 10 ou superior (64-bit)
  • Pelo menos 8GB de RAM
  • Pelo menos 4GB de espaço em disco livre
  • Conexão estável com a internet

📥 Passo 1: Instalar o Flutter SDK

Baixar o Flutter SDK

Baixar Flutter SDK para Windows (Versão Estável Mais Recente)

O que é isso? O Flutter SDK é como a caixa de ferramentas para criar o aplicativo. Ele contém tudo que você precisa para desenvolver, testar e construir o app. Usamos a versão mais recente para garantir que você tenha as últimas funcionalidades e segurança.

Adicionar ao PATH

  1. Pressione Win + R, digite sysdm.cpl e pressione Enter
  2. Clique na aba Avançado
  3. Clique em Variáveis de Ambiente
  4. Na seção Variáveis do sistema, selecione Path e clique em Editar
  5. Clique em Novo e adicione: C:\src\flutter\bin
  6. Clique OK para fechar todas as janelas

Importante: Após realizar esta etapa, você precisa fechar e abrir novamente qualquer terminal (Prompt de Comando, PowerShell, etc.) para que a alteração tenha efeito.

🔍 Passo 2: Verificar Dependências

Executar flutter doctor

Abra um novo terminal e execute:

flutter doctor

Se você vir um erro como "'flutter' não é reconhecido", vá para a seção Solução de Problemas.

🤖 Passo 3: Instalar Android Studio

Baixar Android Studio

Baixar Android Studio

Por que isso é necessário? O Android Studio é o programa oficial do Google para desenvolver apps Android. Mesmo que você use outro editor (como VS Code), ele é essencial porque fornece ferramentas para o Flutter construir e testar o aplicativo no Android.

Configurar Android Studio

  1. Abra o Android Studio.
  2. No assistente de instalação, certifique-se de marcar:
    • Android SDK
    • Android SDK Platform
    • Android Virtual Device
  3. Clique em Next e complete a instalação.

✅ Passo 5: Finalizar Configuração

Aceitar licenças Android

Execute no Prompt de Comando ou PowerShell:

flutter doctor --android-licenses

Pressione y para aceitar todas as licenças.

Verificar instalação

Execute novamente:

flutter doctor

Agora deverá mostrar todas as dependências como OK.

❓ Solução de Problemas: Comando 'flutter' não reconhecido

Este erro ocorre quando o sistema não encontra o programa Flutter. É comum e fácil de resolver! Ele acontece porque o caminho para o Flutter não foi configurado corretamente nas variáveis de ambiente do seu computador.

Windows

Cenário do Erro

Você abre o terminal (Prompt de Comando ou PowerShell), digita flutter doctor e vê:

'flutter' não é reconhecido como um comando interno
ou externo, um programa operável ou um arquivo em lotes.

Como Resolver

  1. Verifique o Caminho do Flutter:
    Confirme onde você extraiu o Flutter SDK. No nosso guia, usamos C:\src\flutter. O caminho a adicionar é a subpasta bin: C:\src\flutter\bin.
  2. Corrija as Variáveis de Ambiente:
    • Pressione Win + R, digite sysdm.cpl e pressione Enter.
    • Vá para a aba Avançado e clique em Variáveis de Ambiente.
    • Na seção Variáveis do sistema, selecione Path e clique em Editar.
    • Verifique se C:\src\flutter\bin está na lista. Se não, clique em Novo e adicione-o.
    • Confirme que não há erros de digitação.
    • Clique em OK para salvar.

    Nota: Se o erro persistir, verifique se o Flutter SDK foi extraído corretamente e se você tem permissões de escrita na pasta C:\src\flutter. Tente executar o terminal como administrador.

  3. Reinicie o Terminal:
    Feche todas as janelas do Prompt de Comando ou PowerShell e abra uma nova. Isso é essencial!
  4. Tente Novamente:
    No novo terminal, execute:
    flutter doctor
    O comando agora deve funcionar e verificar sua configuração.

macOS

Verificar e Corrigir

  1. Confirme que o Flutter SDK está em ~/development/flutter.
  2. Abra o terminal e edite o arquivo de perfil (~/.zshrc ou ~/.bashrc):
    nano ~/.zshrc
  3. Adicione a linha:
    export PATH="$PATH:~/development/flutter/bin"
  4. Salve e atualize o perfil:
    source ~/.zshrc
  5. Teste com:
    flutter doctor

Nota: Se o erro persistir, verifique as permissões da pasta ~/development/flutter e tente executar como administrador com sudo.

Linux/ChromeOS

Verificar e Corrigir

  1. Confirme que o Flutter SDK está em ~/flutter.
  2. Abra o terminal e edite ~/.bashrc ou ~/.zshrc:
    nano ~/.bashrc
  3. Adicione a linha:
    export PATH="$PATH:~/flutter/bin"
  4. Salve e atualize:
    source ~/.bashrc
  5. Teste com:
    flutter doctor

Nota: Se o erro persistir, verifique as permissões da pasta ~/flutter e tente executar como administrador com sudo.

❓ Solução de Problemas: Erro de Licenças Android

Se o flutter doctor indicar que as licenças Android não estão aceitas, siga estes passos simples:

Como Resolver

  1. Execute no terminal:
    flutter doctor --android-licenses
  2. Pressione y para aceitar todas as licenças.
  3. Reexecute flutter doctor para verificar.

Nota: Certifique-se de que o Android Studio está instalado, pois ele fornece as ferramentas necessárias para as licenças.

Voltar para a Configuração

❓ Solução de Problemas: Problemas com Emulador

Se o emulador Android não iniciar ou o Flutter não o detectar, siga estes passos:

Como Resolver

  1. Abra o Android Studio e vá para Configure > AVD Manager.
  2. Crie um novo dispositivo virtual (escolha um dispositivo como Pixel 4 e uma API recente).
  3. Inicie o emulador e verifique se ele aparece ao executar:
    flutter devices
  4. Se não for detectado, verifique se o Android SDK está configurado corretamente no Android Studio.

Nota: Emuladores exigem um computador com bom desempenho. Considere testar em um dispositivo físico se o emulador for lento.
