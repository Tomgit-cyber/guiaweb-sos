📘 Visão Geral do Projeto

Este documento apresenta a visão macro do projeto, detalhando seus objetivos, escopo e principais componentes.

🎯 Objetivo Principal

Desenvolver um aplicativo móvel de código aberto e multiplataforma que permita aos associados da Unimed acionar rapidamente o serviço de emergência, facilitando o acesso a cuidados médicos urgentes.

📱 Escopo do MVP

  • Funcionalidade Chave: Botão de emergência para ligar para a Unimed.
  • Autenticação: Login seguro para associados.
  • Perfil do Usuário: Visualização de dados básicos.
  • Histórico de Chamadas: Registro das chamadas realizadas.

⚙️ Configuração no ChromeOS/Linux

Siga este guia para configurar seu ambiente de desenvolvimento Flutter no ChromeOS com suporte a Linux (Crostini).

📋 Pré-requisitos

  • Chromebook com suporte a Linux (Crostini)
  • Pelo menos 8GB de armazenamento livre
  • Conexão estável com a internet

🔧 Passo 1: Configurar o Ambiente Linux

Ativar Linux (Crostini)

  1. Acesse Configurações > Linux (Desenvolvedor)
  2. Clique em Ativar
  3. Siga as instruções na tela (pode demorar alguns minutos)

Atualizar pacotes

Abra o Terminal Linux e execute:

sudo apt-get update && sudo apt-get upgrade -y

📥 Passo 2: Instalar o Flutter

Baixar o Flutter SDK

cd ~
wget https://storage.googleapis.com/flutter_infra_release/releases/stable/linux/flutter_linux_3.22.0-stable.tar.xz

Extrair o arquivo

tar xf flutter_linux_3.22.0-stable.tar.xz

Adicionar ao PATH

  1. Edite o arquivo .bashrc:
    2. nano ~/.bashrc
  2. Adicione no final:
    3. export PATH="$PATH:$HOME/flutter/bin"
  3. Salve (Ctrl+O, Enter) e saia (Ctrl+X)
  4. Carregue as alterações:
    5. source ~/.bashrc

🔍 Passo 3: Verificar Dependências

Executar flutter doctor

flutter doctor

Você verá mensagens sobre dependências faltando - vamos resolver isso agora.

🤖 Passo 4: Configurar Android

Instalar Java JDK

sudo apt-get install openjdk-11-jdk -y

Baixar Android Studio

  1. Acesse developer.android.com/studio
  2. Baixe a versão para Linux
  3. No terminal, navegue até a pasta de downloads e extraia:
    4. cd ~/Downloads
    unzip android-studio-*-linux.tar.gz
    mv android-studio ~

Iniciar Android Studio

cd ~/android-studio/bin
./studio.sh

Siga o assistente de instalação e marque:

  • Android SDK
  • Android SDK Platform
  • Android Virtual Device

Configurar variáveis de ambiente

  1. Edite novamente o .bashrc:
    2. nano ~/.bashrc
  2. Adicione:
    3. export ANDROID_HOME="$HOME/Android/Sdk"
    export PATH="$PATH:$ANDROID_HOME/tools"
    export PATH="$PATH:$ANDROID_HOME/platform-tools"
  3. Salve e carregue:
    4. source ~/.bashrc

✅ Passo 5: Finalizar Configuração

Aceitar licenças Android

flutter doctor --android-licenses

Pressione y para aceitar todas as licenças.

Verificar instalação

flutter doctor

Agora deverá mostrar todas as dependências como OK.

⚙️ Configuração no Windows

Siga este guia para configurar seu ambiente de desenvolvimento Flutter no Windows.

📋 Pré-requisitos

  • Windows 10 ou superior (64-bit)
  • Pelo menos 8GB de RAM
  • Pelo menos 4GB de espaço em disco livre
  • Conexão estável com a internet

📥 Passo 1: Instalar o Flutter SDK

Baixar o Flutter SDK

  1. Acesse docs.flutter.dev/get-started/install/windows
  2. Baixe o arquivo flutter_windows_x.x.x-stable.zip
  3. Extraia o arquivo em C:\src\flutter (crie a pasta se necessário)

Adicionar ao PATH

  1. Pressione Win + R, digite sysdm.cpl e pressione Enter
  2. Clique na aba Avançado
  3. Clique em Variáveis de Ambiente
  4. Na seção Variáveis do sistema, selecione Path e clique em Editar
  5. Clique em Novo e adicione: C:\src\flutter\bin
  6. Clique OK para fechar todas as janelas

🔍 Passo 2: Verificar Dependências

Executar flutter doctor

Abra o Prompt de Comando ou PowerShell e execute:

flutter doctor

Você verá mensagens sobre dependências faltando - vamos resolver isso agora.

🤖 Passo 3: Instalar Android Studio

Baixar Android Studio

  1. Acesse developer.android.com/studio
  2. Baixe a versão para Windows
  3. Execute o instalador e siga as instruções

Configurar Android Studio

  1. Abra o Android Studio
  2. No assistente de instalação, marque:
    • Android SDK
    • Android SDK Platform
    • Android Virtual Device
  3. Clique em Next e complete a instalação

📱 Passo 4: Configurar Emulador Android (Opcional)

Criar um dispositivo virtual

  1. No Android Studio, vá em Tools > Device Manager
  2. Clique em Create Device
  3. Selecione um dispositivo (ex: Pixel 4)
  4. Selecione uma imagem do sistema (API 30 ou superior recomendado)
  5. Clique em Finish

✅ Passo 5: Finalizar Configuração

Aceitar licenças Android

Execute no Prompt de Comando ou PowerShell:

flutter doctor --android-licenses

Pressione y para aceitar todas as licenças.

Verificar instalação

Execute novamente:

flutter doctor

Agora deverá mostrar todas as dependências como OK.

⚙️ Configuração no macOS

Siga este guia para configurar seu ambiente de desenvolvimento Flutter no macOS. Esta etapa é **obrigatória** para compilar e implantar o app para iOS.

📋 Pré-requisitos

  • macOS (High Sierra ou superior)
  • Pelo menos 8GB de RAM
  • Pelo menos 4GB de espaço em disco livre
  • Conexão estável com a internet

📥 Passo 1: Instalar o Flutter SDK

Baixar o Flutter SDK

  1. Acesse docs.flutter.dev/get-started/install/macos
  2. Baixe o arquivo flutter_macos_arm64_x.x.x-stable.zip (para Apple Silicon) ou flutter_macos_x64_x.x.x-stable.zip (para Intel)
  3. Extraia o arquivo na sua pasta Applications ou em um diretório de sua escolha (ex: ~/development)

Adicionar ao PATH

  1. Determine qual shell você está usando: echo $SHELL
  2. Se for zsh (padrão no macOS Catalina e posteriores), edite ~/.zshrc. Se for bash, edite ~/.bash_profile.
  3. Adicione a seguinte linha (ajuste o caminho se necessário):
    4. export PATH="$PATH:$HOME/development/flutter/bin"
  4. Salve o arquivo e execute source ~/.zshrc (ou source ~/.bash_profile) para carregar as alterações.

🔍 Passo 2: Instalar e Configurar o Xcode

Instalar Xcode

  1. Abra a App Store
  2. Procure por "Xcode" e instale. A instalação pode demorar bastante.
  3. Após a instalação, abra o Xcode e aceite os termos de licença.
  4. Execute os seguintes comandos no Terminal para garantir que as ferramentas de linha de comando estejam configuradas:
    5. sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer
    sudo xcodebuild -runFirstLaunch

✅ Passo 3: Finalizar Configuração

Rodar Flutter Doctor

Abra um novo Terminal e execute novamente:

flutter doctor

Siga as instruções para resolver qualquer pendência, como a instalação do CocoaPods (`sudo gem install cocoapods`) e a aceitação das licenças do Android.

🔐 Arquitetura de Perfis - Visão Geral

O aplicativo SOS Unimed Vidas segue uma arquitetura de navegação condicional baseada no perfil do usuário após o login. Isso significa que um único app pode oferecer funcionalidades distintas para diferentes tipos de usuários.

👥 Tipos de Usuários

  • Paciente: Usuário final que utilizará o serviço de emergência. Tem acesso a funcionalidades como acionar emergência, visualizar perfil e histórico de chamadas.
  • Administrador: Usuário com acesso privilegiado. Tem acesso a funcionalidades de gestão, como dashboards, gerenciamento de pacientes e visualização de alertas.

🏗️ Estrutura Geral

SOS Unimed Vidas (App Único)
├── Tela de Login
│   ├── Validação de credenciais no Firebase Auth
│   └── (Opcional) Seleção de tipo de usuário (Paciente/Admin)
├── Tela Principal (Home)
│   ├── Para Pacientes:
│   │   ├── Informações pessoais
│   │   ├── Botão de emergência médica
│   │   ├── Histórico de atendimentos
│   │   └── Perfil e configurações
│   └── Para Administradores:
│       ├── Dashboard com estatísticas (Total de pacientes, atendimentos, alertas)
│       ├── Gerenciamento de pacientes
│       ├── Atendimentos em andamento
│       ├── Alertas pendentes
│       └── Ferramentas de teste (ex: Testar Conexão Firestore)
└── Navegação Condicionada por Perfil

Importante: Telas administrativas (como dashboards) não devem ser acessadas por pacientes — elas são exclusivas para administradores e protegidas por regras de segurança.

Próximo: Implementação

🔐 Arquitetura de Perfis - Implementação

Esta seção detalha como implementar o sistema de perfis no aplicativo Flutter.

1. Estrutura no Firebase Authentication + Firestore

Após o cadastro do usuário no Firebase Authentication, armazene os dados adicionais no Firestore, incluindo o campo role.

{ "uid": "abc123", "email": "paciente@unimed.com", "name": "João da Silva", "role": "patient", // ou "admin" "createdAt": "2025-04-01T10:00:00Z" }

2. Verificação de Perfil Após Login

Após o login bem-sucedido, consulte o Firestore para obter o role do usuário e redirecione para a tela inicial apropriada.

// Exemplo em Dart (Flutter) Future loginAndRedirect(String email, String password) async { try { final userCredential = await AuthService.signIn(email, password); final userData = await FirestoreService.getUserData(userCredential.user!.uid); if (userData['role'] == 'admin') { Navigator.pushReplacementNamed(context, '/admin/home'); } else { Navigator.pushReplacementNamed(context, '/patient/home'); } } catch (e) { // Tratar erro de login print("Erro no login: $e"); } }

3. Estrutura de Pastas no Projeto

Organize o código para refletir a separação de responsabilidades por perfil.

🔐 Arquitetura de Perfis - Segurança

Implementar regras rigorosas no Firebase Firestore é crucial para proteger os dados e garantir que usuários só acessem informações para as quais têm permissão.

Regras de Segurança no Firestore

Configure as regras no Firebase Console (Cloud Firestore > Rules) para controlar o acesso com base no role do usuário.

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Função para verificar se o usuário é admin function isAdmin() { return request.auth != null && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin'; } // Pacientes só podem ler seus próprios dados match /patients/{patientId} { allow read: if request.auth != null && request.auth.uid == resource.data.uid; // Escrita pode ser permitida com validações adicionais allow write: if request.auth != null && request.auth.uid == resource.data.uid; } // Apenas administradores podem acessar dados de pacientes e áreas administrativas match /patients/{patientId} { allow read, write, update, delete: if isAdmin(); } match /admin/{document=**} { allow read, write: if isAdmin(); } // Adicione regras para outros documentos conforme necessário } }

✅ Checklist de Implementação de Perfis

  • Criar campo role no cadastro de usuários no Firestore.
  • Implementar lógica de verificação de perfil após autenticação.
  • Criar rotas condicionais (/patient/home, /admin/home).
  • Desenvolver telas específicas para perfil de paciente.
  • Desenvolver telas específicas para perfil de administrador.
  • Configurar regras de segurança no Firestore para controlar acesso por perfil.
  • Testar login/logout com diferentes perfis de usuário.
  • Documentar o fluxo de autenticação e navegação condicional no guia.
