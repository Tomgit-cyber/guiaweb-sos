🎯 Bem-vindo ao Guia de Desenvolvimento

ℹ️ Sobre este Guia

Este guia é específico para o desenvolvimento do aplicativo SOS Unimed Vidas utilizando o framework Flutter. Ele fornece instruções passo a passo para configurar o ambiente, criar o projeto, implementar funcionalidades e preparar o MVP para entrega.

Escopo: O foco está em criar um aplicativo multiplataforma (Android, iOS, Web) com Flutter. Este NÃO é um guia geral de desenvolvimento web ou mobile. As instruções são práticas e diretas ao objetivo do projeto.

Este guia orienta você em cada etapa do desenvolvimento do aplicativo SOS Unimed Vidas, desde a configuração inicial até a entrega do MVP funcional.

Como usar este guia:

  1. Use o menu à esquerda para navegar pelas fases do desenvolvimento.
  2. Clique em cada item para ver instruções detalhadas.
  3. Consulte a seção Solução de Problemas caso encontre dificuldades.
  4. Siga os passos na ordem para melhores resultados.
  5. Marque as tarefas como concluídas para acompanhar seu progresso.

O objetivo é criar um aplicativo funcional com documentação clara para a equipe de TI da Unimed continuar o projeto.

Começar pelo Primeiro Passo

🖥️ Configuração Inicial: Windows

Vamos configurar seu ambiente de desenvolvimento Flutter no Windows.

📥 Passo 1: Baixar e Extrair Flutter SDK

Baixar Flutter SDK

Baixar Flutter SDK para Windows

Importante: Baixe a versão estável mais recente.

Extrair SDK

  1. Extraia o arquivo ZIP baixado para um local conhecido, como D:\flutter ou C:\src\flutter.
  2. Evite extrair para pastas com espaços ou caracteres especiais no nome (ex: "Arquivos de Programas").
  3. Após a extração, verifique se o arquivo flutter\bin\flutter.bat existe no local escolhido.

⚙️ Passo 2: Configurar PATH do Sistema

Adicionar Flutter ao PATH

  1. Pressione Win + R, digite sysdm.cpl e pressione Enter.
  2. Na aba Avançado, clique em Variáveis de Ambiente.
  3. Em Variáveis do sistema, selecione Path e clique em Editar.
  4. Clique em Novo e adicione o caminho completo para a pasta bin do Flutter (ex: D:\flutter\bin).
  5. Clique em OK para salvar todas as alterações.

Nota: Feche e reabra o terminal (Prompt de Comando ou PowerShell) para aplicar as mudanças.

🔍 Passo 3: Verificar Dependências

Executar flutter doctor

Abra um novo terminal e execute:

flutter doctor

Se aparecer um erro como "'flutter' não é reconhecido", verifique:

  • O caminho no PATH está correto (ex.: `D:\flutter\bin`).
  • Reinicie o terminal após adicionar o PATH.
  • Consulte Solução de Problemas para mais detalhes.

🤖 Passo 4: Instalar Android Studio

Baixar Android Studio

Baixar Android Studio

Importante: O Android Studio é necessário para desenvolver apps Android.

Instalar Android Studio

  1. Execute o instalador baixado.
  2. Siga as instruções na tela.
  3. Durante a instalação, certifique-se de marcar as opções para instalar o Android SDK, Android SDK Platform-Tools e Android SDK Build-Tools.
  4. Após a instalação, inicie o Android Studio.
  5. Siga o assistente de configuração para instalar as plataformas Android necessárias (SDK).

📜 Passo 5: Aceitar Licenças do Android

Aceitar licenças

Execute no terminal:

flutter doctor --android-licenses

Pressione y para aceitar cada licença solicitada.

✅ Passo 6: Verificar Instalação

Confirmar configuração

Execute novamente:

flutter doctor

Confirme que todas as dependências estão marcadas como OK.

📋 Passo 1: Conhecer a Estrutura

Estrutura de Pastas

Após criar o projeto, a pasta sos_unimed_vidas contém:

🗂️ Passo 2: Configurar Dependências

Atualizar pubspec.yaml

Adicione as dependências necessárias ao pubspec.yaml:

dependencies:
flutter:
sdk: flutter
http: ^0.13.5
provider: ^6.0.5
firebase_core: ^2.1.1
firebase_auth: ^4.1.1
cloud_firestore: ^4.0.4

Próximo: Lógica de Negócio

🧠 Lógica de Negócio

Desenvolver a lógica principal do aplicativo, incluindo autenticação e gerenciamento de chamadas de emergência.

🔑 Passo 1: Implementar Autenticação

Autenticação com Firebase Auth

Em lib/services/auth_service.dart, crie métodos para:

  • Registro de usuário (signUp)
  • Login (signIn)
  • Logout (signOut)
  • Verificação do estado de autenticação (isLoggedIn)
// Exemplo em lib/services/auth_service.dart
import 'package:firebase_auth/firebase_auth.dart';

class AuthService {
final FirebaseAuth _auth = FirebaseAuth.instance;

User? getCurrentUser() {
return _auth.currentUser;
}

Future<User?> signInWithEmailAndPassword(String email, String password) async {
try {
UserCredential result = await _auth.signInWithEmailAndPassword(email: email, password: password);
return result.user;
} catch (e) {
print(e.toString());
return null;
}
}

Future<void> signOut() async {
await _auth.signOut();
}
}

Integre esses métodos nas telas de login e registro.

📞 Passo 2: Gerenciamento de Chamadas

Modelo de Chamada e Serviço

Crie um modelo lib/models/call.dart para representar uma chamada de emergência.

Em lib/services/call_service.dart, crie métodos para:

  • Registrar nova chamada (createCall)
  • Atualizar status da chamada (updateCallStatus)
  • Obter chamadas de um usuário (getCallsForUser)
// Exemplo em lib/models/call.dart
class Call {
String id;
String userId;
String type;
String status;
DateTime timestamp;
// outros campos

Call({required this.id, required this.userId, required this.type, required this.status, required this.timestamp});

Map<String, dynamic> toMap() {
return {
'userId': userId,
'type': type,
'status': status,
'timestamp': timestamp.toIso8601String(),
};
}

static Call fromMap(Map<String, dynamic> map, String id) {
return Call(
id: id,
userId: map['userId'],
type: map['type'],
status: map['status'],
timestamp: DateTime.parse(map['timestamp']),
);
}
}

// Exemplo em lib/services/call_service.dart
import 'package:cloud_firestore/cloud_firestore.dart';
import 'package:sos_unimed_vidas/models/call.dart';

class CallService {
final FirebaseFirestore _firestore = FirebaseFirestore.instance;

Future<void> createCall(Call call) async {
await _firestore.collection('calls').add(call.toMap());
}

Stream<List<Call>> getCallsByUserId(String userId) {
return _firestore.collection('calls')
.where('userId', isEqualTo: userId)
.orderBy('timestamp', descending: true)
.snapshots()
.map((snapshot) => snapshot.docs.map((doc) => Call.fromMap(doc.data(), doc.id)).toList());
}
}

🔄 Passo 3: Gerenciamento de Estado

Utilizar Provider

Use o pacote provider para gerenciar o estado global do aplicativo, como o usuário logado e o status das chamadas.

Exemplo no main.dart:

import 'package:provider/provider.dart';
import 'package:sos_unimed_vidas/services/auth_service.dart';

void main() async {
WidgetsFlutterBinding.ensureInitialized();
await Firebase.initializeApp();
runApp(MyApp());
}

class MyApp extends StatelessWidget {
@override
Widget build(BuildContext context) {
return MultiProvider(
providers: [
Provider<AuthService>(create: (_) => AuthService()),
// Adicione outros providers aqui, como CallService
],
child: MaterialApp(
// ... tema e outras configurações
home: Consumer<AuthService>(
builder: (context, authService, _) {
return StreamBuilder<User?>(
stream: FirebaseAuth.instance.authStateChanges(),
builder: (context, snapshot) {
if (snapshot.connectionState == ConnectionState.active) {
User? user = snapshot.data;
if (user == null) {
return LoginScreen();
} else {
return HomeScreen();
}
} else {
return const Scaffold(body: Center(child: CircularProgressIndicator()));
}
},
);
},
),
),
);
}
}

import 'package:firebase_core/firebase_core.dart';

O arquivo firebase_options.dart é gerado automaticamente pelo FlutterFire CLI após você configurar o Firebase. Se não tiver, execute:

✍️ Passo 2: Criar Serviço de API

Implementar Chamadas GET/POST

Em lib/services/api_service.dart, crie métodos para interagir com sua API.

// Exemplo em lib/services/api_service.dart
import 'dart:convert';
import 'package:http/http.dart' as http;

class ApiService {
final String _baseUrl = 'https://sua-api.com/api/v1'; // Substitua pela sua URL

Future<dynamic> getRequest(String endpoint) async {
final response = await http.get(Uri.parse('$_baseUrl/$endpoint'));
return _processResponse(response);
}

Future<dynamic> postRequest(String endpoint, Map<String, dynamic> data) async {
final response = await http.post(
Uri.parse('$_baseUrl/$endpoint'),
headers: {'Content-Type': 'application/json'},
body: json.encode(data),
);
return _processResponse(response);
}

dynamic _processResponse(http.Response response) {
if (response.statusCode == 200 || response.statusCode == 201) {
return json.decode(response.body);
} else {
throw Exception('Falha na requisição: ${response.statusCode}');
}
}
}

📊 Passo 3: Utilizar na Lógica de Negócio

Integrar com Telas/Providers

Chame os métodos do ApiService nas suas telas ou provedores para buscar e exibir dados.

// Exemplo em um Provider ou Widget
class MyDataProvider extends ChangeNotifier {
final ApiService _apiService = ApiService();
List<dynamic> _data = [];

List<dynamic> get data => _data;

Future<void> fetchData() async {
try {
_data = await _apiService.getRequest('items');
notifyListeners();
} catch (e) {
print('Erro ao buscar dados: $e');
}
}
}

Próximo: Solução de Problemas

❓ Solução de Problemas: Comando 'flutter' não reconhecido

Este problema ocorre quando o diretório do Flutter SDK não está no PATH do sistema.

🛠️ Solução para Windows:

Verificar e Adicionar ao PATH

  1. Confirme o caminho onde você extraiu o Flutter SDK (ex: D:\flutter).
  2. O caminho que precisa ser adicionado ao PATH é D:\flutter\bin (substitua D:\flutter pelo seu caminho).
  3. Siga os passos do "Passo 2: Configurar PATH do Sistema" na seção Configuração Inicial: Windows.
  4. Importante: Após alterar as variáveis de ambiente, feche todas as instâncias do Prompt de Comando/PowerShell e abra uma nova para que as mudanças surtam efeito.

🛠️ Solução para macOS/Linux:

Verificar e Atualizar Shell Profile

  1. Confirme onde você extraiu o Flutter SDK (ex: ~/development/flutter).
  2. O caminho a ser adicionado é ~/development/flutter/bin.
  3. Siga os passos do "Passo 2: Configurar PATH" na seção Configuração Inicial: macOS ou Configuração Inicial: ChromeOS/Linux.
  4. Após editar o arquivo .bashrc ou .zshrc, execute source ~/.bashrc (ou source ~/.zshrc) ou reinicie o terminal.

Próximo: Erro de licenças Android

❓ Solução de Problemas: Erro de licenças Android

Esse erro geralmente significa que você não aceitou todas as licenças do Android SDK.

🛠️ Solução: Aceitar Licenças

Executar `flutter doctor --android-licenses`

  1. Abra um terminal (Prompt de Comando, PowerShell ou Terminal do macOS/Linux).
  2. Execute o seguinte comando:
    3. flutter doctor --android-licenses
  3. Quando solicitado, digite y e pressione Enter para aceitar cada licença. Repita até que não haja mais licenças a serem aceitas.
  4. Após aceitar todas as licenças, execute flutter doctor novamente para confirmar que o erro foi resolvido.

Se o problema persistir, pode ser necessário abrir o Android Studio, ir em SDK Manager e verificar se todos os componentes necessários estão instalados e atualizados.

Próximo: Problemas com Emulador

❓ Solução de Problemas: Problemas com Emulador

Se seu emulador Android não inicia ou não é detectado pelo Flutter.

🛠️ Solução 1: Iniciar Emulador Manualmente

Abrir AVD Manager

  1. Abra o Android Studio.
  2. No menu, vá em Tools > Device Manager (ou AVD Manager em versões mais antigas).
  3. Selecione o emulador que deseja usar e clique no botão de "Play" para iniciá-lo.
  4. Aguarde o emulador carregar completamente.
  5. No terminal, execute flutter devices para verificar se ele é detectado.

🛠️ Solução 2: Criar um Novo Emulador

Criar um AVD (Android Virtual Device) Novo

  1. No Android Studio, vá em Tools > Device Manager.
  2. Clique em "Create device".
  3. Escolha uma definição de hardware (ex: Pixel 6) e clique em "Next".
  4. Selecione uma imagem de sistema (API Level, ex: Android S - API 31 ou 32) e clique em "Download" se necessário.
  5. Clique em "Next", dê um nome ao AVD e finalize.
  6. Tente iniciar o novo emulador.

🛠️ Solução 3: Verificar Instalação do HAXM (Windows/macOS) ou KVM (Linux)

Aceleração de Hardware

  • Windows/macOS: Certifique-se de que o HAXM (Intel Hardware Accelerated Execution Manager) está instalado e configurado corretamente. Você pode instalá-lo via SDK Manager no Android Studio ou baixá-lo diretamente do site da Intel. Verifique se a virtualização está habilitada na BIOS/UEFI do seu computador.
  • Linux: Verifique se o KVM (Kernel-based Virtual Machine) está habilitado e configurado. Instale os pacotes necessários (ex: sudo apt install qemu-kvm libvirt-daemon-system libvirt-clients bridge-utils) e adicione seu usuário aos grupos apropriados (ex: sudo adduser $USER libvirt, sudo adduser $USER kvm).
