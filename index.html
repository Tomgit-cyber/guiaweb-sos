🎯 Bem-vindo ao Guia de Desenvolvimento

Este guia foi criado para te orientar em cada etapa do desenvolvimento do aplicativo SOS Unimed Vidas, desde a configuração inicial até a entrega final do MVP.

Como usar este guia:

  1. Use o menu à esquerda para navegar pelas fases do desenvolvimento.
  2. Se encontrar um problema, verifique a seção Solução de Problemas.
  3. Use o Histórico de Erros para registrar e acompanhar issues.
  4. Siga os passos na ordem recomendada para melhor resultado.

O objetivo é entregar um MVP completo e funcional, com todas as etapas documentadas para facilitar a continuidade do projeto pela equipe de TI da Unimed.

Começar Pela Configuração

⚙️ Configuração no Windows

1. Baixar o Flutter SDK

Este é o kit de desenvolvimento principal do Flutter, necessário para criar, testar e compilar o aplicativo.

Baixar Flutter SDK para Windows

2. Adicionar ao PATH

Esta etapa informa ao Windows onde encontrar o Flutter, permitindo que você execute comandos como `flutter doctor` de qualquer local no terminal.

  1. Pressione a tecla Windows + R para abrir a caixa "Executar".
  2. Digite sysdm.cpl e pressione Enter.
  3. Na janela "Propriedades do Sistema", clique na aba Avançado.
  4. Clique no botão Variáveis de Ambiente....
  5. Na seção inferior ("Variáveis do sistema"), encontre a variável Path, selecione-a e clique em Editar....
  6. Na nova janela, clique em Novo.
  7. Cole o caminho completo para a pasta `bin` do Flutter (ex: C:\src\flutter\bin).
  8. Clique em OK em todas as janelas para salvar.

Importante: Após realizar esta etapa, você precisa fechar e abrir novamente qualquer terminal para que a alteração tenha efeito.

3. Executar Flutter Doctor

Este comando verifica se o ambiente está totalmente configurado. Abra um novo terminal e execute:

flutter doctor

Se o comando não for reconhecido, veja a solução aqui.

4. Baixar o Android Studio

O Android Studio é essencial pois instala as ferramentas (SDK) que o Flutter usa para criar a versão Android do aplicativo.

Baixar Android Studio

❓ Solução: Comando 'flutter' não reconhecido

Cenário do Erro

Você abre o terminal, digita flutter doctor e recebe a seguinte mensagem:

'flutter' não é reconhecido como um comando interno ou externo...

Causa Principal

O sistema operacional não sabe onde encontrar o executável do Flutter. Isso ocorre porque o caminho para a pasta `bin` do SDK não foi adicionado corretamente às Variáveis de Ambiente do Windows.

✅ Como Resolver (Passo a Passo Detalhado)

  1. Confirme o Caminho: Usando o Explorador de Arquivos, navegue até a pasta onde você extraiu o Flutter. Entre na pasta `bin`. Clique na barra de endereço na parte superior e copie este caminho. Ele deve ser algo como `C:\src\flutter\bin`.
  2. Abra as Propriedades do Sistema: Pressione a tecla **Windows + R**. Na caixa "Executar", digite `sysdm.cpl` e clique em OK.
  3. Acesse as Variáveis de Ambiente: Na janela que abriu, vá para a aba **Avançado** e clique no botão **Variáveis de Ambiente...**.
  4. Edite a Variável Path: Na seção de baixo ("Variáveis do sistema"), encontre e selecione a linha que diz `Path`. Em seguida, clique em **Editar...**.
  5. Adicione o Caminho do Flutter: Na nova janela, clique em **Novo** no lado direito. Um novo campo de texto aparecerá no final da lista. Cole o caminho que você copiou no passo 1 (ex: `C:\src\flutter\bin`).
  6. Salve Tudo: Clique em **OK** para fechar a janela de edição do Path. Clique em **OK** novamente para fechar as Variáveis de Ambiente. Clique em **OK** uma última vez para fechar as Propriedades do Sistema.
  7. Reinicie o Terminal (Crucial): Feche TODAS as janelas do Prompt de Comando ou PowerShell que estiverem abertas. Abra uma nova janela de terminal. Esta nova sessão agora conhecerá o novo caminho.
  8. Verifique Novamente: No novo terminal, digite `flutter doctor`. O comando deve funcionar.

💻 1. Criar o Projeto Flutter

Executar Comando de Criação

Este comando cria uma nova pasta com toda a estrutura básica de um projeto Flutter.

flutter create sos_unimed_vidas

Navegar para a Pasta

Entre na pasta do projeto recém-criado para executar os próximos comandos.

cd sos_unimed_vidas

📦 2. Build e Assinatura (Android)

1. Criar uma Chave de Assinatura

Esta chave é um certificado digital único que prova que você é o autor do aplicativo. É obrigatória para publicar na Google Play Store.

keytool -genkey -v -keystore C:\Users\SEU_USUARIO\upload-keystore.jks -keyalg RSA -keysize 2048 -validity 10000 -alias upload

Substitua `SEU_USUARIO` pelo seu nome de usuário do Windows. O terminal pedirá para você criar uma senha e preencher algumas informações. Guarde a senha em um local seguro.

2. Configurar a Referência da Chave

Agora, precisamos dizer ao projeto onde encontrar a chave e a senha que você acabou de criar.

  1. Dentro do seu projeto Flutter, navegue até a pasta `android`.
  2. Dentro da pasta `android`, crie um novo arquivo de texto e nomeie-o exatamente como `key.properties`.
  3. Abra este novo arquivo e adicione as seguintes linhas, substituindo os valores conforme necessário:
storePassword=SUA_SENHA_DA_CHAVE
keyPassword=SUA_SENHA_DA_CHAVE
keyAlias=upload
storeFile=C:\\Users\\SEU_USUARIO\\upload-keystore.jks

Atenção: Use barras duplas (`\\`) no caminho do `storeFile` como mostrado acima.

3. Gerar o Aplicativo (APK)

Este comando compila o código e empacota o aplicativo em um arquivo `.apk` assinado e pronto para instalação.

flutter build apk --release

O arquivo final estará em `build\app\outputs\flutter-apk\app-release.apk`.
